Pre-flight

Checklista przed uruchomieniem

Runbook

Przebieg demonstracji krok po kroku

1. Start
Uruchom launch, sprawdź obrazy i stan diagnostyk.
2. Obserwacja
Śledź przejścia: SEARCH_TABLE → ALIGN_WITH_BOX → PICK_BOX.
3. Interwencja
Przy odchyleniach użyj pause; przy zagrożeniu użyj e-stop.
Operator powinien reagować na timeouty stanów ALIGN_WITH_BOX i NAVIGATE_TO_TARGET_MARKER zanim robot zakończy demonstrację w stanie błędu lub utknie na scenie.

Najczęstszy punkt interwencji: ALIGN_WITH_BOX. Jeżeli offset nie maleje lub marker jest tracony, użyj pause i popraw geometrię sceny albo kalibrację kamery.
W stanie NAVIGATE_TO_TARGET_MARKER obserwuj odległość do znacznika i pamiętaj, że odometria jest przybliżona. To pomoc operatorska, nie źródło precyzyjnej lokalizacji.

Odzyskiwanie

Procedury po błędzie

Utrata markera
Pause, przywróć widoczność tagu, resume lub reset.
Timeout FSM
Sprawdź /diagnostics i wykonaj reset przed nowym przebiegiem.
Ryzyko kolizji
Natychmiast e-stop, potem ocena fizyczna robota.
# 1. Emergency stop tylko jeśli istnieje ryzyko ruchowe ros2 service call /robomvp/emergency_stop std_srvs/Trigger "{}" # 2. Po ustabilizowaniu sytuacji sprawdź łączność ping 192.168.123.161 # 3. Zresetuj logikę ros2 service call /robomvp/reset std_srvs/Trigger "{}" # 4. Uruchom launch ponownie ros2 launch robomvp demo.launch.py network_interface:=eth0 require_robot_connection:=true
